# Contributing to VARYNX 2.0

Thank you for your interest in VARYNX 2.0.

---

## Bug Reports

Found a bug? Please open an issue on our [GitHub Issues](../../issues) page with:

- A clear description of the problem
- Steps to reproduce (if known)
- Which platform you're running (Android, Desktop, Linux, HomeHub, Pocket, Satellite, WearOS)
- OS version and device model
- Expected behavior vs. actual behavior

---

## Feature Requests

Have an idea? Open a [GitHub Issue](../../issues) and tag it as a feature request. Include:

- What the feature would do
- Why it matters to you
- Which platform(s) it applies to

We review all feature requests and prioritize based on security impact and community demand.

---

## Security Vulnerabilities

**Do NOT report security vulnerabilities through public GitHub issues.**

If you discover a vulnerability, please report it privately:

- **Email**: [varynx.contact@gmail.com](mailto:varynx.contact@gmail.com)
- See our [Security Policy](SECURITY.md) for full details

We take all security reports seriously and will respond within 48 hours.

---

## Code Contributions

VARYNX 2.0 is **proprietary software**. We do not accept external code contributions at this time.

This project is the product of careful, security-focused engineering. Every line of code is authored, reviewed, and tested by the VARYNX team. Accepting external code in a security-critical application introduces risk that we are not willing to take.

---

## Community

The best ways to support VARYNX:

- Report bugs and request features through [GitHub Issues](../../issues)
- Share VARYNX with others who value device security and privacy
- Star the repo if you find the project valuable
